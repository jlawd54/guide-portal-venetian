# guide-portal-venetian

## 项目简介

guide-portal-venetian 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为 HTML 静态资源的集中存储与版本管理之用。

## 目录结构

```
guide-portal-venetian/
├── pages/            # 存放独立 HTML 页面文件
├── assets/           # 页面引用的公共资源（CSS、JS、图片等）
├── index.html        # 仓库根目录入口页面（可选）
└── README.md         # 本文件
```

- `pages/`：每个子目录或文件代表一个独立页面，文件名或目录名反映页面主题。
- `assets/`：存放页面间共享的静态资源，按类型或用途组织子目录。

## 页面归档说明

本仓库中的 HTML 页面均为独立、自包含的静态页面。每个页面均可直接通过浏览器打开，不依赖后端服务或特定域名解析。页面内容可能涉及信息展示、工具界面、文档说明等，均以 HTML 标准编写。

页面归档遵循以下原则：

- 每个页面保持独立，避免交叉依赖（公共资源除外）。
- 页面文件名或路径应反映其内容或用途，便于查找。
- 归档内容不包含任何商业推广、诱导点击或具体域名指向。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/guide-portal-venetian.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件，或通过本地 HTTP 服务器预览。
3. 如需部署到静态托管服务，可将整个仓库内容上传至服务根目录。

## 维护说明

- **添加页面**：在 `pages/` 下新建 HTML 文件或目录，确保页面可独立运行。
- **更新资源**：修改 `assets/` 中的公共资源时，注意检查引用该资源的所有页面是否需要同步调整。
- **删除页面**：移除对应文件或目录，并清理不再使用的公共资源引用。
- **版本管理**：遵循 Git 常规流程，提交信息应简要说明变更内容。

## 贡献

欢迎提交 Pull Request 来完善或新增页面。请确保：

- 页面内容不违反任何法律法规。
- 页面不含恶意代码或跟踪脚本。
- 新增资源尽量复用 `assets/` 中已有文件，避免冗余。
